```
// プレイヤーのデータ
```

### 💾 データを保存する問題

**めたん**：「コンピュータのメモリ上にあるデータは、プログラムが動いてる間だけ存在するんです」

**つむぎ**：「あー、ゲーム終了したら消えちゃうってこと？」

**めたん**：「そうです。だから...」

1. **ファイルに保存したい** → セーブデータ
2. **サーバーに送りたい** → オンラインゲーム
3. **他のプレイヤーと共有したい** → マルチプレイ

**ずんだもん**：「どれも必要なのだ！」

**めたん**：「そこで、メモリ上のデータをファイルやネットワークで送れる形に変換する必要があるんです」

**つむぎ**：「それが『シリアライゼーション』ってやつ？」

**めたん**：「正解です！」

---

### 🔄 シリアライゼーションの流れ

**ずんだもん**：「図で説明するのだ！」

```
【保存するとき】
メモリ上のデータ（オブジェクト）
    ↓ シリアライゼーション
バイト列（0101010...）
    ↓
ファイルに保存 or ネットワークで送信

【読み込むとき】
ファイルから読み込み or ネットワークで受信
    ↓
バイト列（0101010...）
    ↓ デシリアライゼーション
メモリ上のデータ（オブジェクト）
```

**つむぎ**：「あー、なるほど！データを『運べる形』にするってことね！」

**ずんだもん**：「お弁当箱に詰めるみたいなものなのだ！」

## 🆚 いろんな形式がある

**めたん**：「シリアライゼーションには、いろんな『形式』があるんです」

**つむぎ**：「形式？」

**ずんだもん**：「お弁当箱にも、竹かご、プラスチック容器、木箱...いろいろあるのだ！」

**めたん**：「そうですね。代表的な形式を紹介します」

### 📋 主なシリアライゼーション形式

**めたん**：「比較表を作りました」

| 形式 | 読みやすさ | サイズ | 速度 | 用途 |
|------|----------|--------|------|------|
| **JSON** | ◎ 人間が読める | △ 大きい | △ 遅い | Web API、設定ファイル |
| **XML** | ○ 人間が読める | ✕ とても大きい | ✕ とても遅い | 古いシステム |
| **MessagePack** | ✕ バイナリ | ◎ 小さい | ◎ 超高速 | ゲーム、リアルタイム通信 |
| **Protobuf** | ✕ バイナリ | ◎ 小さい | ○ 速い | Google製、サーバー間通信 |

**つむぎ**：「MessagePackってめっちゃ速いじゃん！」

**ずんだもん**：「サイズも小さいのだ！すごいのだ！」

### 📄 具体例：JSONとの比較

**めたん**：「同じデータを、JSONとMessagePackで保存した例を見てみましょう」

#### JSON形式

```json
{
  "name": "ずんだもん",
```

**つむぎ**：「これ、普通に読めるね！」

#### MessagePack形式

```
85 A4 6E 61 6D 65 AF E3 81 9A E3 82 93 E3 81 A0
E3 82 82 E3 82 93 A5 6C 65 76 65 6C 63 A2 68 70

```

**つむぎ**：「うわ、なにこれ！？全然読めない！」

**ずんだもん**：「暗号みたいなのだ！」

**めたん**：「これがバイナリ形式です。人間には読めませんが...」
- サイズ：約80バイト（**JSONの半分！**）
- 読み書き速度：**JSONの3〜5倍速い！**

**つむぎ**：「マジで！？それヤバくない！？」

## ⚡ MessagePackの特徴

**ずんだもん**：「MessagePackの凄いところをまとめるのだ！」

### ✨ MessagePackの強み

**めたん**：「主に4つあります」

#### 1️⃣ めちゃくちゃ速い

**つむぎ**：「どれくらい速いの？」

**めたん**：「ベンチマークでは...」
- JSON（Json.NET）の **3〜5倍速い**
- Unity標準のJsonUtilityより **10倍以上速い**

**ずんだもん**：「爆速なのだ！」

**つむぎ**：「それ、具体的にどういう時に嬉しいの？」

**めたん**：「例えば...」
- オンラインゲームのリアルタイム通信
- 大量のデータを保存・読み込み
- モバイルゲームの起動時間短縮

**つむぎ**：「あー、起動遅いとイライラするやつ！」

---

#### 2️⃣ データサイズが小さい

**ずんだもん**：「JSONの半分以下のサイズなのだ！」

**つむぎ**：「それって何が嬉しいの？」

**めたん**：「こういう効果があります」
- **通信量削減** → スマホの通信料節約
- **ストレージ節約** → セーブデータが軽い
- **読み込み高速化** → データ量が少ない分、速い

**つむぎ**：「特にスマホゲームには重要そう！」

---

#### 3️⃣ C#に最適化されている

**めたん**：「MessagePack for C#は、Unityでの使用を前提に作られているんです」

**ずんだもん**：「Unityで使いやすいのだ！」

**つむぎ**：「具体的には？」

**めたん**：「こんな特徴があります」
- IL2CPP（Unity のビルド）に完全対応
- AOT（事前コンパイル）対応
- Zero Allocation（メモリ効率が良い）
- コード自動生成で高速化

**つむぎ**：「よくわかんないけど、Unityに優しいってことね！」

---

#### 4️⃣ 型安全

**めたん**：「C#の型システムをそのまま使えるんです」

**つむぎ**：「型？」

**ずんだもん**：「データの種類のことなのだ！」

**めたん**：「例えば、こういうクラスをそのまま保存できます」

```csharp
[MessagePackObject]
public class PlayerData
{
    [Key(0)]
    public string Name { get; set; }

    [Key(1)]
    public int Level { get; set; }

    [Key(2)]

    [Key(3)]
    public List<string> Items { get; set; }

```

**つむぎ**：「お、これなら見やすい！」

**ずんだもん**：「普通のC#のクラスなのだ！」

**めたん**：「属性（`[MessagePackObject]`など）を付けるだけで、自動でシリアライズできるんです」

## 💻 基本的な使い方

**つむぎ**：「じゃあ、実際にどう使うのか教えてよ！」

**ずんだもん**：「コード見せるのだ！」

### ステップ1：クラスを定義

**めたん**：「まず、保存したいデータをクラスとして定義します」

```csharp
using MessagePack;

[MessagePackObject]
public class SaveData
{
    [Key(0)]
    public string PlayerName { get; set; }

    [Key(1)]
    public int Level { get; set; }

    [Key(2)]
    public int HP { get; set; }

    [Key(3)]
    public int Gold { get; set; }
}
```

**つむぎ**：「`[MessagePackObject]`と`[Key(0)]`って何？」

**めたん**：「これは『属性』といって、MessagePackに『このクラスはシリアライズできるよ』と教えるものです」

**ずんだもん**：「目印みたいなものなのだ！」

---

### ステップ2：データを作る

```csharp
// セーブデータを作成
var data = new SaveData
{
    PlayerName = "ずんだもん",
    Level = 99,
    HP = 9999,
    Gold = 123456
};
```

**つむぎ**：「普通にクラスのインスタンス作ってるだけだね！」

---

### ステップ3：シリアライズ（保存）

**めたん**：「データをバイト列に変換して保存します」

```csharp
// バイト列に変換
byte[] bytes = MessagePackSerializer.Serialize(data);

// ファイルに保存
File.WriteAllBytes("savedata.dat", bytes);
```

**ずんだもん**：「たった2行なのだ！簡単なのだ！」

**つむぎ**：「マジで？こんな簡単なの！？」

---

### ステップ4：デシリアライズ（読み込み）

**めたん**：「保存したデータを読み込むのも簡単です」

```csharp
// ファイルから読み込み
byte[] bytes = File.ReadAllBytes("savedata.dat");

// オブジェクトに復元
SaveData data = MessagePackSerializer.Deserialize<SaveData>(bytes);

// 使える！
Debug.Log(data.PlayerName); // "ずんだもん"
Debug.Log(data.Level);      // 99
```

**つむぎ**：「おー！元に戻った！」

**ずんだもん**：「魔法みたいなのだ！」

## 🎮 実際の使用例

**つむぎ**：「で、実際にどんな場面で使うの？」

**めたん**：「いくつか例を紹介しますね」

### 例1：セーブデータ

**ずんだもん**：「一番よくある使い方なのだ！」

```csharp
[MessagePackObject]
public class GameSaveData
{
    [Key(0)]
    public string PlayerName { get; set; }

    [Key(1)]
    public int Level { get; set; }

    [Key(2)]
    public Vector3 Position { get; set; }  // Unityの型もOK！

    [Key(3)]
    public DateTime LastSaveTime { get; set; }

    [Key(4)]
    public List<InventoryItem> Inventory { get; set; }
}

// 保存
public void SaveGame()
{
    var data = new GameSaveData { /* ... */ };
    var bytes = MessagePackSerializer.Serialize(data);
    File.WriteAllBytes("save.dat", bytes);
}

// 読み込み
public GameSaveData LoadGame()
{
    var bytes = File.ReadAllBytes("save.dat");
    return MessagePackSerializer.Deserialize<GameSaveData>(bytes);
}
```

**つむぎ**：「シンプルでいいね！」

---

### 例2：オンラインゲームの通信

**めたん**：「リアルタイム通信にも最適です」

```csharp
[MessagePackObject]
public class PlayerMovePacket
{
    [Key(0)]
    public int PlayerId { get; set; }

    [Key(1)]
    public Vector3 Position { get; set; }

    [Key(2)]
    public Quaternion Rotation { get; set; }
}

// サーバーに送信
public void SendPosition(Vector3 pos, Quaternion rot)
{
    var packet = new PlayerMovePacket
    {
        PlayerId = myId,
        Position = pos,
        Rotation = rot
    };

    byte[] data = MessagePackSerializer.Serialize(packet);
    networkClient.Send(data);  // 超軽量＆高速！
}
```

**つむぎ**：「これ、データ小さいからラグも少なそう！」

**ずんだもん**：「オンライン対戦ゲームにぴったりなのだ！」

---

### 例3：マスターデータの読み込み

**めたん**：「ゲームの設定データにも使えます」

```csharp
[MessagePackObject]
public class EnemyMasterData
{
    [Key(0)]
    public int Id { get; set; }

    [Key(1)]
    public string Name { get; set; }

    [Key(2)]
    public int HP { get; set; }

    [Key(3)]
    public int Attack { get; set; }

// 大量のデータでも高速読み込み

{
    var bytes = Resources.Load<TextAsset>("EnemyMaster").bytes;

}
```

**つむぎ**：「敵キャラ1000体分のデータとかでも一瞬で読めそう！」

**ずんだもん**：「ゲーム起動が速くなるのだ！」

## 🆚 他の選択肢との比較

**つむぎ**：「じゃあさ、いつMessagePack使って、いつJSON使えばいいの？」

**めたん**：「いい質問ですね。状況別に整理しましょう」

### 📊 使い分けガイド

**ずんだもん**：「表にしたのだ！」

| 状況 | おすすめ | 理由 |
|------|---------|------|
| **設定ファイル** | JSON | 人間が編集する必要がある |
| **デバッグ用データ** | JSON | 内容を確認したい |
| **セーブデータ** | MessagePack | 速度とサイズ重視 |
| **オンライン通信** | MessagePack | 通信量削減が重要 |
| **マスターデータ** | MessagePack | 起動速度重視 |
| **Web API** | JSON | 標準的で互換性が高い |
| **データ解析用** | JSON/CSV | 後から見直す可能性 |

**つむぎ**：「あー、なるほど！人が見るならJSON、速度重視ならMessagePackって感じ？」

**めたん**：「基本的にはそうですね」

**ずんだもん**：「使い分けが大事なのだ！」

**めたん**：「クラスの構造を変えると、古いデータが読めなくなることがあります」

```csharp

    public string Name { get; set; }
    [Key(1)]

public class SaveData

    public string Name { get; set; }
```
