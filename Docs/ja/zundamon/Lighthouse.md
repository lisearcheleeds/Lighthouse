# 🎮 【Unity初心者向け】Lighthouse Architectureって何？

## 📱 まずは身近な例から

**つむぎ**：「ねえねえ、スマホでゲームやアプリ使ってる？」

**ずんだもん**：「使ってるのだ！」

**つむぎ**：「じゃあさ、こういう経験ない？」
- タイトル画面 → メニュー画面 → ゲーム画面って切り替わる
- ボタン押したらダイアログが出てくる
- 設定で言語を日本語⇔英語に変える

**めたん**：「そういった機能、実は全部プログラムで作られているんです」

**ずんだもん**：「当たり前なのだ！」

**つむぎ**：「でもさ、毎回ゼロから作るのマジでダルくない？」

**めたん**：「ええ、そこで登場するのがLighthouse Architectureなんです」

---

## 🏗️ Lighthouse Architectureって何？

**ずんだもん**：「一言で言うと...ゲームやアプリを作るときの便利な土台なのだ！」

**つむぎ**：「例えばさ、家を建てるとき、毎回レンガから作る？」

**めたん**：「いえ、普通は既に作られた材料を使いますよね」

**つむぎ**：「そう！Lighthouse Architectureは、よく使う機能をあらかじめ用意してくれてる便利セットって感じ！」

**ずんだもん**：「プラモデルみたいなものなのだ！部品が最初から揃ってるのだ！」

---

## 🎯 具体的に何ができるの？

**つむぎ**：「じゃあ、具体的に何ができるか見てこっか！」

### 1️⃣ 画面の切り替え（シーン管理）

**ずんだもん**：「タイトル画面からメニュー画面に移動する、みたいなやつなのだ！」

**めたん**：「普通にプログラムを書くと、こういう問題が起きます」
- 前の画面が消えない
- アニメーションがカクカク
- 戻るボタンが効かない

**つむぎ**：「あーしも最初作った時、マジでバグりまくったわ...」

**ずんだもん**：「でもLighthouseを使えば、そういうの全部自動でやってくれるのだ！」

**つむぎ**：「画面の切り替えがヌルヌル動くし、戻るボタンもちゃんと動くのよ！マジ便利！」

---

### 2️⃣ ダイアログの管理

**つむぎ**：「次はダイアログ！ほら、「本当に削除しますか？」みたいなやつ」

**ずんだもん**：「ポップアップ画面なのだ！」

**めたん**：「ダイアログを何個も重ねたり、順番に閉じたりするのは意外と難しいんです」

**つむぎ**：「例えばさ...」

```
ゲーム画面
  └→ メニューを開く
      └→ 設定画面を開く
          └→ 確認ダイアログを開く
```

**つむぎ**：「こうやって重なってる時、どの順番で閉じる？背景タップした時どうする？」

**ずんだもん**：「難しそうなのだ...」

**つむぎ**：「でもLighthouseなら、スタックって仕組みで自動管理してくれるの！」

**めたん**：「スタックは「積み重ね」という意味です。本を積み重ねるように、ダイアログを管理するんですね」

**ずんだもん**：「一番上の本から取るように、一番上のダイアログから閉じるのだ！簡単なのだ！」

---

### 3️⃣ 多言語対応（日本語・英語など）

**つむぎ**：「これマジで便利！あーし的には一番推し！」

**ずんだもん**：「世界中の人に遊んでもらえるのだ！」

**めたん**：「普通に多言語対応しようとすると...」

```csharp
// 悪い例
if (language == "ja") {
    text = "こんにちは";
} else if (language == "en") {
    text = "Hello";
} else if (language == "zh") {
    text = "你好";
}
```

**つむぎ**：「これ、100個テキストあったらマジで地獄でしょ？」

**ずんだもん**：「コードがめちゃくちゃ長くなるのだ...」

**めたん**：「でもLighthouseなら、翻訳データをExcelみたいな表で管理できるんです」

**つむぎ**：「こんな感じ！」

| キー | 日本語 | 英語 | 中国語 |
|------|--------|------|--------|
| GREETING | こんにちは | Hello | 你好 |
| START | スタート | Start | 开始 |
| QUIT | 終了 | Quit | 退出 |

**ずんだもん**：「これなら見やすいのだ！」

**つむぎ**：「しかもボタン一つで言語切り替えできるし、フォントも自動で変わるの！マジ神！」

## 🧩 どうやって作られてるの？

**めたん**：「少し技術的な話になりますが、わかりやすく説明しますね」

**ずんだもん**：「Lighthouseは、有名なライブラリを組み合わせて作られてるのだ！」

### 使われている技術

**つむぎ**：「レゴブロックみたいに、いろんなパーツを組み合わせてるの！」

#### 🔹 VContainer（ブイコンテナ）

**めたん**：「部品を自動で組み立ててくれる仕組みです」

**つむぎ**：「え、なにそれ？」

**ずんだもん**：「例えばさ、車を作るとき...」
- エンジン
- タイヤ
- ハンドル

**ずんだもん**：「これを全部手で組み立てるのは大変なのだ！」

**めたん**：「VContainerは、『車を作って』と言えば、必要な部品を自動で集めて組み立ててくれるんです」

**つむぎ**：「あ、なるほど！自動組み立て機みたいな？」

**めたん**：「そうですね、そんな感じです」

---

#### 🔹 UniTask（ユニタスク）

**ずんだもん**：「時間がかかる処理を楽に書けるやつなのだ！」

**つむぎ**：「時間がかかる処理って？」

**めたん**：「例えば、こういうのです」
1. 画面をフェードアウト（1秒かかる）
2. データを読み込む（2秒かかる）
3. 画面をフェードイン（1秒かかる）

**つむぎ**：「あー、読み込み中のローディング画面とか！」

**めたん**：「ええ。普通に書くと複雑になるんですが、UniTaskを使うと...」

```csharp
// こんな感じでスッキリ書ける
await FadeOut();      // フェードアウト待つ
await LoadData();     // 読み込み待つ
await FadeIn();       // フェードイン待つ
```

**ずんだもん**：「awaitって書くだけで、待ってくれるのだ！簡単なのだ！」

**つむぎ**：「おー、マジでわかりやすい！」

---

#### 🔹 R3（アールスリー）

**めたん**：「値が変わった時に自動で反応してくれる仕組みです」

**つむぎ**：「？？？」

**ずんだもん**：「例を出すのだ！」

**めたん**：「例えば、HPが変わった時、HPバーも自動で更新したいですよね？」

**つむぎ**：「あー、RPGとかの体力ゲージね！」

**めたん**：「普通だと、HP変更するたびに『HPバー更新して！』って書かないといけません」

```csharp
// 面倒な書き方
HP = HP - 10;
UpdateHPBar();  // ←これ書き忘れそう

HP = HP + 20;
UpdateHPBar();  // ←毎回書かないといけない
```

**つむぎ**：「確かに、これ書き忘れたらバグるわ...」

**めたん**：「でもR3を使うと、HPが変わったら自動的にHPバーも更新されるんです」

**ずんだもん**：「便利なのだ！」

**つむぎ**：「それマジで楽じゃん！」
